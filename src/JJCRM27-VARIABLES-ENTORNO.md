### 🔍 DETALLES POR VARIABLE

#### `SUPABASE_URL`
- **Usado en**: `/supabase/functions/server/index.tsx`, `/supabase/functions/server/kv_store.tsx`
- **Función**: Conectar Edge Functions a la base de datos Supabase
- **Formato**: `https://[PROJECT_ID].supabase.co`
- **Dónde obtenerla**: Supabase Dashboard → Settings → API → Project URL

#### `SUPABASE_SERVICE_ROLE_KEY`
- **Usado en**: `/supabase/functions/server/index.tsx`, `/supabase/functions/server/kv_store.tsx`
- **Función**: Autenticar operaciones del servidor con permisos completos
- **Formato**: JWT largo (eyJhbGciOi...)
- **Dónde obtenerla**: Supabase Dashboard → Settings → API → Service Role Key (secret)
- **⚠️ CRÍTICO**: NO compartir públicamente, solo en variables de entorno del servidor

#### `GOOGLE_MAPS_API_KEY`
- **Usado en**: `/supabase/functions/server/index.tsx` → Endpoint `/api-keys/google-maps`
- **Función**: Cálculo de distancias, geocodificación, mapas en módulos Carroll y Cotizaciones
- **Módulos que la usan**:
  - CotizacionesModule (distancias entre rutas)
  - MapaFlotaGoogleMaps (visualización flota)
  - UbicacionGPS (reverse geocoding)
- **Dónde obtenerla**: Google Cloud Console → APIs & Services → Credentials

#### `OPENWEATHER_API_KEY`
- **Usado en**: `/supabase/functions/server/index.tsx` → Endpoint `/api-keys/openweather`
- **Función**: Capas meteorológicas en Mapa Climático Carroll
- **Módulos que la usan**:
  - MapaClimaticoCarroll (temperatura, precipitación, vientos)
- **Dónde obtenerla**: https://openweathermap.org/api

#### PASO 2: Crear Edge Function de Deployment

**Archivo**: `/supabase/functions/server/deployment.tsx`

```typescript
import { Hono } from "npm:hono";

const deploymentApp = new Hono();

// POST: Desplegar JJCRM27
deploymentApp.post("/make-server-d84b50bb/deploy/jjcrm27", async (c) => {
  try {
    const { usuario } = await c.req.json();
    
    // 1. Validar que el usuario sea administrador
    if (usuario?.rol !== 'administrador') {
      return c.json({ 
        success: false, 
        error: 'Solo administradores pueden desplegar' 
      }, 403);
    }

    const token = Deno.env.get('GITHUB_TOKEN_JJCRM27');
    const repo = Deno.env.get('REPO_JJCRM27') || 'vivercan/izzi';
    const branch = Deno.env.get('BRANCH_JJCRM27') || 'main';

    if (!token) {
      return c.json({ 
        success: false, 
        error: 'GITHUB_TOKEN_JJCRM27 no configurado' 
      }, 500);
    }

    // 2. Crear commit vacío con mensaje de deployment
    // (El trigger de Vercel se activa con cualquier push)
    const timestamp = new Date().toISOString();
    const commitMessage = `🚀 Deploy JJCRM27 - ${timestamp}`;
    
    // 3. Usar GitHub API para crear commit
    // Referencia: https://docs.github.com/en/rest/git/refs
    
    // 3.1 Obtener SHA del último commit en main
    const refResponse = await fetch(
      `https://api.github.com/repos/${repo}/git/ref/heads/${branch}`,
      {
        headers: {
          'Authorization': `Bearer ${token}`,
          'Accept': 'application/vnd.github+json',
        }
      }
    );
    
    if (!refResponse.ok) {
      throw new Error(`Error obteniendo ref: ${await refResponse.text()}`);
    }
    
    const refData = await refResponse.json();
    const lastCommitSha = refData.object.sha;
    
    // 3.2 Obtener árbol del commit
    const commitResponse = await fetch(
      `https://api.github.com/repos/${repo}/git/commits/${lastCommitSha}`,
      {
        headers: {
          'Authorization': `Bearer ${token}`,
          'Accept': 'application/vnd.github+json',
        }
      }
    );
    
    if (!commitResponse.ok) {
      throw new Error(`Error obteniendo commit: ${await commitResponse.text()}`);
    }
    
    const commitData = await commitResponse.json();
    const treeSha = commitData.tree.sha;
    
    // 3.3 Crear nuevo commit vacío (sin cambios en archivos)
    const newCommitResponse = await fetch(
      `https://api.github.com/repos/${repo}/git/commits`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${token}`,
          'Accept': 'application/vnd.github+json',
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          message: commitMessage,
          tree: treeSha,
          parents: [lastCommitSha],
        })
      }
    );
    
    if (!newCommitResponse.ok) {
      throw new Error(`Error creando commit: ${await newCommitResponse.text()}`);
    }
    
    const newCommitData = await newCommitResponse.json();
    const newCommitSha = newCommitData.sha;
    
    // 3.4 Actualizar la referencia de la rama main
    const updateRefResponse = await fetch(
      `https://api.github.com/repos/${repo}/git/refs/heads/${branch}`,
      {
        method: 'PATCH',
        headers: {
          'Authorization': `Bearer ${token}`,
          'Accept': 'application/vnd.github+json',
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          sha: newCommitSha,
        })
      }
    );
    
    if (!updateRefResponse.ok) {
      throw new Error(`Error actualizando ref: ${await updateRefResponse.text()}`);
    }
    
    console.log(`[DEPLOY JJCRM27] ✅ Commit creado: ${newCommitSha}`);
    console.log(`[DEPLOY JJCRM27] ✅ Push exitoso a ${repo}/${branch}`);
    console.log(`[DEPLOY JJCRM27] ✅ Vercel auto-deploy iniciado`);
    
    return c.json({ 
      success: true, 
      message: 'Deploy iniciado exitosamente',
      commit: newCommitSha,
      timestamp,
      vercelUrl: 'https://www.jjcrm27.com',
      note: 'El deploy tomará 2-3 minutos. Vercel lo desplegará automáticamente.'
    });

  } catch (error) {
    console.error('[DEPLOY JJCRM27] Error:', error);
    return c.json({ 
      success: false, 
      error: String(error) 
    }, 500);
  }
});

export default deploymentApp;
```

#### PASO 4: Crear componente de botón en UI

**Archivo**: `/components/fx27/DeployButton.tsx`

```typescript
import { useState } from 'react';
import { Rocket, Loader2, CheckCircle, XCircle } from 'lucide-react';
import { projectId, publicAnonKey } from '../../utils/supabase/info';

interface DeployButtonProps {
  usuario: {
    rol: string;
    nombre: string;
  };
}

export function DeployButton({ usuario }: DeployButtonProps) {
  const [deploying, setDeploying] = useState(false);
  const [status, setStatus] = useState<'idle' | 'success' | 'error'>('idle');
  const [message, setMessage] = useState('');

  // Solo mostrar para administradores
  if (usuario.rol !== 'administrador') {
    return null;
  }

  const handleDeploy = async () => {
    if (!confirm('¿Estás seguro de desplegar JJCRM27 a producción?\n\nEsto actualizará www.jjcrm27.com en 2-3 minutos.')) {
      return;
    }

    setDeploying(true);
    setStatus('idle');
    setMessage('Iniciando deployment...');

    try {
      const response = await fetch(
        `https://${projectId}.supabase.co/functions/v1/make-server-d84b50bb/deploy/jjcrm27`,
        {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${publicAnonKey}`,
            'Content-Type': 'application/json',
          },
          body: JSON.stringify({ usuario })
        }
      );

      const data = await response.json();

      if (data.success) {
        setStatus('success');
        setMessage(`✅ Deploy iniciado exitosamente!\n\n🔗 Commit: ${data.commit.substring(0, 7)}\n⏱️ Tiempo estimado: 2-3 minutos\n🌐 URL: ${data.vercelUrl}\n\n${data.note}`);
      } else {
        setStatus('error');
        setMessage(`❌ Error: ${data.error}`);
      }
    } catch (error) {
      setStatus('error');
      setMessage(`❌ Error de conexión: ${error}`);
    } finally {
      setDeploying(false);
    }
  };

  return (
    <div className="fixed bottom-8 right-8 z-50">
      <button
        onClick={handleDeploy}
        disabled={deploying}
        className="group relative flex items-center gap-3 px-6 py-4 bg-gradient-to-r from-blue-600 to-blue-700 text-white rounded-xl shadow-lg hover:shadow-xl hover:scale-105 transition-all duration-300 disabled:opacity-50 disabled:cursor-not-allowed"
      >
        {deploying ? (
          <Loader2 className="w-5 h-5 animate-spin" />
        ) : status === 'success' ? (
          <CheckCircle className="w-5 h-5 text-green-300" />
        ) : status === 'error' ? (
          <XCircle className="w-5 h-5 text-red-300" />
        ) : (
          <Rocket className="w-5 h-5 group-hover:translate-y-[-2px] transition-transform" />
        )}
        <span className="font-semibold" style={{ fontFamily: 'Exo 2' }}>
          {deploying ? 'Desplegando...' : 'Deploy JJCRM27 🚀'}
        </span>
      </button>

      {/* Mensaje de status */}
      {message && (
        <div className={`absolute bottom-full right-0 mb-4 max-w-md p-4 rounded-lg shadow-xl ${
          status === 'success' ? 'bg-green-50 border border-green-200' :
          status === 'error' ? 'bg-red-50 border border-red-200' :
          'bg-blue-50 border border-blue-200'
        }`}>
          <p className="text-sm whitespace-pre-line" style={{ 
            fontFamily: 'Exo 2',
            color: status === 'success' ? '#166534' : status === 'error' ? '#991b1b' : '#1e40af'
          }}>
            {message}
          </p>
          <button
            onClick={() => setMessage('')}
            className="mt-2 text-xs underline opacity-70 hover:opacity-100"
          >
            Cerrar
          </button>
        </div>
      )}
    </div>
  );
}
```

## 5️⃣ TROUBLESHOOTING

### ❌ Error: "GITHUB_TOKEN_JJCRM27 no configurado"

**Causa**: Variable de entorno faltante  
**Solución**:
1. Generar token en GitHub (Settings → Developer Settings → Personal Access Tokens)
2. Agregar en Vercel → Settings → Environment Variables
3. Redeploy el Edge Function

### ❌ Error: "Solo administradores pueden desplegar"

**Causa**: Usuario no tiene rol correcto  
**Solución**:
1. Verificar en `localStorage.getItem('fx27-usuarios')`
2. Actualizar rol del usuario a `'administrador'`
3. Refrescar sesión

### ❌ Error: "Error actualizando ref"

**Causa**: Permisos insuficientes del GitHub Token  
**Solución**:
1. Regenerar token con scope `repo` completo
2. Actualizar variable `GITHUB_TOKEN_JJCRM27`
3. Reintentar deployment

### ❌ Deployment tarda más de 5 minutos

**Causa**: Vercel puede estar saturado o hay error en build  
**Solución**:
1. Revisar Vercel Dashboard → Deployments
2. Ver logs del build
3. Si hay error, corregir y hacer push manual

### ⚠️ Commits vacíos aparecen en GitHub

**Esperado**: Sí, esto es intencional  
**Razón**: El botón crea commits vacíos solo para disparar Vercel  
**Alternativa**: Modificar la función para hacer build real y subir `dist/`
